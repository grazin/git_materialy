Wróciłeś po dniu nieobecności do pracy, zbliża się deadline, ale nie wypchnąłeś jeszcze swoich zmian z ostatniego dnia w którym byłeś w pracy, czas to nadrobić

Użyj repozytorium z poprzedniego ćwiczenia.

$ git checkout hello-[your-github-username]
$ git branch

Upewnij się, że pracujesz na swoim branczu hello-[your-github-username].

Stwórz jakiś plik, dodaj go do indeksu i zakomituj

$ git push origin hello-[your-github-username]

Teraz możesz wypchąć swoje zmiany do zdalnego repo, tak aby wszyscy mogli podziwiać twoją pracę.

Kolega przypomina ci, że dopóki nie domerdżujesz zmian z brancha important to nie wyrobicie się do deadline

$ git fetch origin important

I już brancza important mamy lokalnie

$ git merge origin important

Teraz wystarczy zmerdżować za pomocą komendy merge

$ git log -5

Sprawdzamy czy faktycznie te zmiany są u nas w branczu.
