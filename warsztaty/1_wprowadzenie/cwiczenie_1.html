Jesteś po wymianie komputera, niestety poprzedni sprzęt uległ kompletnej destrukcji, więc zespół wsparcia technicznego nie miał z czego odzyskać danych. Czas więc postawić środowisko od nowa i poprawnie je skonfigurować

$ git init

Zacznij od utworzenia nowego repozytorium lokalnego za pomocą polecenia init.

$ git config user.name <imie i nazwisko>
$ git config user.email <email>

Zacznij od wprowadzenia danych identyfikacyjnych user.name user.email. Zrobisz to za pomocą komendy config.

$ git config --global alias.aliases "config --get-regexp ^alias\."

Uznałeś, że warto też dodać alias alias, który w prosty sposób pozwoli ci na wyświetlenie wszystkich dostępnych aliasów. Używasz w tym celu polecenia config z parametrem --get-regexp i podająć regexp, który pozwala znaleźć wszystkie aliasy w tekście. A --global pozwoli ustawić ten alias dla wszystkich twoich projektów

$ git config -l --show-origin

Na koniec tylko upewniasz się, że twoja konfiguracja zawiera to co do niej wprowadziłeś. Możesz tego dokonać za pomocą znacznika -l lub --list. A ostatni parametr --show-origin doda tylko informacje, z której konfiguracji jest dany zapis.
