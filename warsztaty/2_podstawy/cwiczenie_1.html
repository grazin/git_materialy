Przychodzisz do pracy po absorbującym weekendzie. Chcesz zakomitować swoje zmiany ale zanim zaczniesz, musisz dowiedzieć się jaki jest stan w twoim repozytorium projektowym.

$ git init

Zacznij od utworzenia nowego repozytorium lokalnego za pomocą polecenia init.

$ git status

Żeby sprawdzić stan, użyj polecenia status. Przeczytaj ze zrozumieniem output dla pustego repozytorium.

$ git add [nazwa_pliku]

Teraz utwórz nowy pusty plik w twoim repozytorium i ponownie sprawdź status. Przeczytaj ze zrozumieniem. A następnie dodaj plik do indeksu. Użyj komendy add i sprawdź status ponownie.

$ git commit -m "Opis tego co zostało zrobione"

Zakomituj teraz nowo utworzony plik. Przeczytaj output z komita.

$ git log

Teraz możesz też już sprawdzić log twojego repozytorium.

$ git add .

Utwórz jeszcze jeden plik, a w pierwszym zmień treść. Sprawdź ponownie status. Dodaj zmiany do indeksu. Tym razem użyj komendy add . czyli "dodaj wszystko". Sprawdź status.

$ git commit

Następnie zakomituj te zmiany, ale nie wpisuj parametru -m.


# Please enter the commit message for your changes. Lines starting
# with '#' will be ignored, and an empty message aborts the commit.
#
# On branch master
# Changes to be commited:
#       modified:  file1.txt
#
~
~

Git bash i tak poprosi cię o wpisanie i zatwierdzenie tytułu komita. Wejdź w tryb edycji i lub Insert. Wprowadź tytul i wyjdź Escape a następnie zakończ Shift+zz lub :wq. Sprawdź status i log
