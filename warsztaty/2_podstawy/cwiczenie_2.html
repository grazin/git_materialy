Zacząłeś pracę nad nową funkcjonalnością w projekcie. Po jakimś czasie masz wątpliwości, czy podejście które zastosowałeś jest najlepsze i chcesz wypróbować inne. Chcesz jednak zachować dotychczasową pracę, gdyby okazało się, że jednak pierwsze podejście było najlepsze.

Użyj repozytorium z poprzedniego ćwiczenia.

$ git stash save

Otwórz jeden z istniejących już plików i dodaj treść. Użyj stosu, żeby odłożyć na ten czas dotychczasową pracę. Zapisz zmiany na stosie za pomocą polecenia stash save.

$ git stash list

Następnie możesz sprawdzić zawartość stosu komendą stash list.

Teraz sprawdź status. Twoje repozytorium jest czyste i jesteś gotów na implementacje nowego podejścia.
Otwórz więc istniejący plik i dodaj/zmień treść.

Okazuje się, że pierwszy sposób był lepszy i chcesz przywrócić teraz zmiany które masz na stosie. Jednak pracy wykonanej także nie chcesz utracić, bo pomysły mogą przydać się w innych okolicznościach.

$ git stash save
$ git stash list

Odłóż więc nowe podejście na stos. Sprawdź stos ponownie.

$ git stash pop stash@{1}

Przywróć docelowe zmiany, które teraz znajdują się na dnie stosu. Twój stasz ma referencję stash@{1}, zdejmij ją ze stosu i usuń za pomocą komendy pop.

$ git stash apply stash@{1}

Jeśli nie chcesz usuwać tych zmian ze stosu zastosuj komendę apply.
