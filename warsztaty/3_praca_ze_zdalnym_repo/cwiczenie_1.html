Te wszystkie zmiany, które udało ci się już wprowadzić, chciałbyś się nimi z kimś podzielić! Tak zdecydowanie to jest to co chciałbyś zrobić

Dlatego sprawdzimy z kim możemy sobie synchronizować dane

$ git remote

No to wyświetlamy zdalne repozytoria za pomocą komendy remote.

Ale, że co, że pusto ??!?

$ git remote add original https://github.com/morelq/cw3.git

No to poprawiamy za pomocą add. Następnie podajemy nazwę a zaraz później url do zdalnego repozytorium.

$ git remote -v

To teraz na pewno mamy jakieś połączenie do zdalnego repozytorium. Sprawdźmy to. Używamy zwykłego remote ale z parametrem -v aby przy okazji sprawdzić czy url się zgadza

Ale zaraz zaraz, jaki original ? Dobre praktyki wskazują, żeby domyślne repozytorium zdalne nazywało się origin, musimy to poprawić

$ git remote rename original origin

Po zastosowaniu rename podając jako pierwszy parametr starą nazwę, a jako drugi parametr nową, wszystko powinno być jak trzeba

$ git config --get-regexp ^remote\.

A co z naszym configiem ? Czy tam też się wszystko zgadza ?

$ git remote show origin

To jeszcze wyświetlmy sobie szczegóły danego połączenia remote show z nazwą repozytorium zwróci odpowiedni rezultat.
