Trzeba kontynuować pracę, ale w momencie kiedy poszedłeś zrobić sobię herbatę, ktoś umyślnie i bez skrupułów udostępnił dwa komity. Co za ludzie

Użyj repozytorium z poprzedniego ćwiczenia.

Nawet jeśli wszyscy już w ciebie zwątpili, pokaż że się ... a nie to nie to.

$ git status

Sprawdzamy stan

$ git fetch origin

Wykonanie komendy fetch origin z parametrem url do zdalnego repozytorium skutkuje stworzeniem kompletnej kopii zdalnego repozytorium.

$ git status

Sprawdzamy stan ponownie

$ git log --oneline master..origin/master

To wypiszmy sobie te komity, które już pobraliśmy ale jeszcze nie mamy ich "zaakceptowanych master..origin/master pozwoli nam wyfiltrować te komity a --oneline jak sama nazwa mówi, wypisze je w jednej lini

$ git log --oneline master..origin/master

To wypiszmy sobie te komity, które już pobraliśmy ale jeszcze nie mamy ich "zaakceptowanych" master..origin/master pozwoli nam wyfiltrować te komity a --oneline jak sama nazwa mówi, wypisze je w jednej lini

$ git pull

Kolejnym krokiem będzie tak jakby od nowa wykonanie operacji fetch ale jednak razem z zaakceptowaniem zmian, i wykorzystamy do tego pull

$ git status

I ponownie sprawdzamy stan

Stwórz teraz nowy katalog niech nosi nazwę: 1 litery imienia i nazwisko napisane razem np: pbartela, a w folderze stwórz plik o dowolnej nazwie i dowolnej zawartości

$ git add .
$ git commit

Następnie dodaj plik do indeksu i zakomituj go

$ git status

I ponownie sprawdzamy stan

$ git push origin

A teraz spróbuj użyć komendy push origin UWAGA, kto pierwszy ten lepszy

Oczywiście żartowałem, ze względu na to, że każdy was będzie wypychał w jednym czasie, musicie używać komend pull i push używać naprzemiennie aż do skutku
